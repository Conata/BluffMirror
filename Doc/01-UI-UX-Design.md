# UI/UX設計仕様

## FPS視点構成

### 画面レイアウト
```
     [AI Hand - 裏向きカード群]
     [AI Face/Symbol - 表情・視線]
====================================== テーブル
     
     [Player Hand - 扇状配置]
                                [HUD - 極小]
```

### メイン画面要素

#### 中央: テーブル
- **素材**: 暗いフェルト
- **照明**: 単一光源による陰影
- **境界**: テーブルエッジで空間を定義
- **演出**: 微細な埃や傷で質感向上

#### 下: プレイヤー手札
- **視点**: 手元視点（自然な角度）
- **配置**: 扇状展開
- **操作**: 指が伸びてカードをつまむモーション
- **フィードバック**: ホバー時の浮き上がり

#### 奥: AI（相手）
- **手札**: 裏向きカード（3-7枚並列）
- **表現**: 手＋顔 または 仮面/シンボル
- **演出**: 表情変化は最小限で不気味さ演出

#### 右下: 極小HUD
- **表示項目**:
  - 残り枚数（数字のみ）
  - ターン表示（シンプル）
  - 捨て札枚数
- **デザイン**: テキストUI最小限

### 画面遷移原則
- **Setup画面なし**: ゲーム設定もテーブル上で完結
- **没入維持**: 画面切り替えによる没入感の断絶を回避
- **物理的操作**: すべての設定をテーブル上の小物で表現

## 操作フロー詳細

### プレイヤーターン（AIから引く）

#### 1. 選択フェーズ
```
AI手札表示（裏向き3-7枚）
↓
カーソル接近
↓
カード浮き上がり（0.5cm + ホバー演出）
↓
心理圧セリフ表示
↓
選択確定（クリック&ドラッグ）
```

#### 2. 引き抜きフェーズ
```
カメラ微寄り（3% dolly-in）
↓
効果音（心音/布擦れ）
↓
AIリアクション（一言セリフ）
↓
カード移動（プレイヤー手札へ吸い込み）
↓
ペア判定・消失演出
```

### AIターン（プレイヤーから引く）

#### 1. AI行動フェーズ
```
AIの手（指）が画面に侵入
↓
プレイヤー手札のターゲット選択
↓
カード引き抜き（表向きで見せられる）
↓
プレイヤーの悔しさ演出
```

### 操作レスポンス目標
- **ホバー反応**: 100ms以内
- **ドラッグ追従**: 16ms（60fps）
- **アニメーション**: 自然な慣性・イージング
- **音響フィードバック**: 操作と同期

## UIポリシー

### 削除すべき従来UI
- ルール説明パネル
- リアルタイム情報ボックス
- ゲームログウィンドウ
- ボタン群
- プログレスバー

### 代替表現方法

#### ルール説明
- **手法**: 開始時にAIが口頭説明＋字幕
- **利点**: キャラクター性向上、没入感維持

#### ゲーム状態表示
- **残り時間**: テーブル上の砂時計
- **残り枚数**: コインの数
- **ターン表示**: 相手の指がテーブルを叩く
- **ゲームログ**: テーブル上の紙片（クリックで展開）

#### ステータス表現
- **指輪**: プレイヤーステータス
- **コイン**: 数値的情報
- **砂時計**: 時間関連情報

## アクセシビリティ配慮

### 視覚的支援
- カードのホバー強調
- 重要情報のハイライト
- コントラスト確保

### 操作支援
- ドラッグエリアの拡大
- 誤操作の取り消し機能
- 段階的なフィードバック

### 音響支援
- 操作音の明確化
- 空間音響による位置認識
- 重要イベントの音響強調
